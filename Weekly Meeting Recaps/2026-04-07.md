## Apr 7, 2026 weekly sync (Jim, Adalee) → this PR

This branch is a **follow-up to our LearnWithMe weekly sync on April 7, 2026**. It doesn’t implement the full roadmap we discussed; it **lands what we can ship now** so we can **keep reviewing the README and Code of Conduct**, tell a **clearer product story**, and give everyone a **cleaner snapshot of the platform on GitHub**.

### How it reflects what we discussed

**README & Code of Conduct**  
We agreed to keep improving how we present the project and our community rules. This PR **refreshes the README**, keeps the **Code of Conduct** easy to read and maintain (`/conduct`), and uses a **shared markdown setup** (`MarkdownDoc`, `remark-gfm` for tables) so long-form docs stay consistent.

**Trust, kids, and expectations**  
We talked about **keeping children in mind** as an org and about **how programs get screened** and how people behave in the product. This PR **surfaces conduct and design principles in the app** (`/conduct`, `/design` + `DESIGN_PRINCIPLES.md`) so expectations aren’t buried. **Content screening, messaging rules, and moderation** are still **follow-up work**, not something this PR claims to finish.

**Kathleen — our first sample member**  
We’re deeply grateful to **Kathleen** for joining us on this journey and helping shape what LearnWithMe can become. The prototype uses her as our **first sample member** so the product shows a real program and profile flow people can explore. **Thank you, Kathleen** — we’re glad you’re here with us, and we can’t wait to keep building alongside you.

**Brand, theme, and marketing**  
We noted we **need a logo** and a **theme for marketing materials**, and a **long-term homepage** rich with instructor photo and video. This PR **does not add a logo or a full marketing site**; it **moves the app toward a single dark visual direction** and **documents design intent** so design and marketing have something concrete to react to. The **instructor-forward homepage** stays on the roadmap.

**Product direction we captured (not built here)**  
The sync captured a lot of **future** work: **per-user progress** through courses, **gamification** (patches, badges, medallions), **visibility toggles** (along the lines of Airbnb), **SSO / Apple Pay / Firebase-style** account stories, **creator-set pricing** and a **platform fee**, **course messaging** (Discord-like, same-course and paying members), **AI moderation**, **preview videos and reviews** (including **post-completion** reviews and **Steam-style “time in course”** ideas), **Zendesk**, **team email**, **where messages live**, and **Notion**. **None of that is in this PR**. Action items like **payments**, **user creation**, and **creator onboarding** stay **tracked separately**.

### What this PR actually changes (technical)

- Site-wide **dark UI** (zinc palette, surfaces, sticky CTA, markdown tables, viewport `theme-color`).
- **`/design`** + **`DESIGN_PRINCIPLES.md`**; **`MarkdownDoc`** replaces the conduct-only markdown component.
- **Copy and navigation** updates across key pages so the prototype matches the **calm, program-first** story we’re telling.
